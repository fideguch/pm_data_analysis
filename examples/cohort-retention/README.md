# Cohort Retention Analysis Example

This example demonstrates pm-data-analysis Capability 4 (Analysis Execution) and Capability 5 (Confidence-Scored Insights) applied to a retention cohort analysis.

## Scenario

A mobile app PM notices declining Android engagement metrics. The analysis uses weekly cohort data split by platform (iOS vs Android) to isolate the issue.

## Key Patterns Demonstrated

- **Platform segmentation** to avoid Simpson's Paradox
- **D1/D7/D30 cascade analysis** showing early-experience root cause
- **7-type bias checklist** with platform-specific notes
- **Confidence scoring** based on sample size, consistency, and effect size
- **Reproduction section** with complete code and re-run instructions

## Files

- `input.csv` — 20 rows of weekly cohort retention data (10 weeks × 2 platforms)
- `analysis_report.md` — Complete analysis output following the skill's report template
